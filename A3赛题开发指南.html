00 为什么选择 OpenMAIC 作为架构参考

OpenMAIC 是什么

OpenMAIC（Open Multi-Agent Interactive Classroom）是清华大学 MAIC 团队开源的多智能体互动课堂系统。它将任意主题转化为完整的交互式课堂体验——包含 AI 教师讲课、幻灯片演示、测验、3D模拟、项目式学习（PBL）等。发表论文于 Journal of Computer Science and Technology (JCST)，MIT 协议，技术架构成熟且前沿。

OpenMAIC 与本次赛题的高度契合

维度 OpenMAIC 已有能力 本次赛题需要扩展的能力
多智能体 LangGraph 编排 AI 教师 + 多位 AI 同学，圆桌讨论、问答模式 增加画像构建Agent、资源生成Agent集群、路径规划Agent
内容生成 一键生成完整课堂（大纲→幻灯片→测验→交互模块→PBL），支持 PPTX 导出 扩展为6种资源类型：文档/导图/题库/视频/代码/阅读材料
交互/渲染 Canvas 幻灯片引擎、SVG 白板、3D可视化、模拟、游戏、在线编程 直接复用交互场景系统，作为"多模态资源"展示层
TTS/多模态 VoxCPM2 语音克隆、多 TTS 提供商、ASR 语音输入 替换为讯飞 TTS，用于教学视频配音和智能辅导朗读
LLM 路由 分阶段模型路由（场景生成/备课/对话用不同模型） 增加讯飞星火作为 Provider，画像/备课/辅导按阶段路由
知识库 PDF/PPTX 文档导入与解析，MinerU 增强解析 扩展为完整 RAG 系统（Chroma + Embedding），支持防幻觉
部署 Docker Compose + Vercel 一键部署 直接复用 Docker 部署方案
🔬 核心复用点
  1. LangGraph 多智能体编排 — OpenMAIC 的状态图编排模式直接适用于"路径规划Agent分配任务给资源生成Agent集群"
  2. 分阶段 LLM 路由 — 画像构建、资源生成、辅导答疑等阶段使用不同模型/参数配置
  3. Scene/Pipeline 两阶段生成 — "大纲→场景内容"的模式直接映射为"学习路径→资源生成"
  4. Canvas 幻灯片 + 交互模块 — 复用作为"多模态资源"的前端渲染方案
  5. Monorepo + pnpm workspaces — 清晰的项目组织方式

架构对比：OpenMAIC vs 我们的系统

OpenMAIC 架构模式

用户输入主题
  → 大纲生成 (outline-generator)
  → 场景生成 (scene-generator)
    ├─ 幻灯片场景 (slide)
    ├─ 测验场景 (quiz)
    ├─ 交互场景 (interactive: 3D/游戏/模拟)
    └─ PBL 场景 (project-based)
  → 课堂播放 (playback state machine)
  → 多智能体讨论 (orchestration graph)

我们的系统 架构模式

用户对话
  → 画像构建 (profile agent)
  → 路径规划 (path agent)
  → 资源生成集群 (resource agents)
    ├─ 讲解文档 (doc — 复用 slide)
    ├─ 思维导图 (mindmap — 复用 diagram)
    ├─ 练习题库 (quiz — 直接复用)
    ├─ 教学视频 (video — 新增 TTS 流程)
    ├─ 代码实操 (code — 复用 online-coding)
    └─ 拓展阅读 (reading — 新增)
  → 智能辅导 (tutor agent)
  → 效果评估 (eval agent)

01 赛题核心要求重述

编号功能模块性质核心要求OpenMAIC 可复用
1对话式学习画像构建 必做 自然语言对话抽取 ≥6 维度画像，随学随更新Chat 对话系统 + Agent 角色系统
2多智能体协同资源生成 必做 不同角色 Agent 协作生成 ≥5 种类型资源Scene 生成 Pipeline + 交互模块
3学习路径规划与推送 必做 智能体协同整合资源，动态规划，精准推送LangGraph 编排 + Playback 状态机
4智能辅导 加分项 即时多模态答疑（文字+图解+短视频）Multi-agent 讨论 + TTS + 白板
5学习效果评估 加分项 实时跟踪、多维度评估、动态调整Quiz 评分系统 + 行为追踪
赛题硬性约束
  • AI 辅助工具必须选用科大讯飞相关工具（星火大模型 API / iFlyCode / 讯飞 TTS）
  • 须明确"多智能体协同框架"
  • 初始知识库需自行构建至少一门完整高校课程
  • 防幻觉与内容安全过滤机制
  • 使用开源项目需标注名称、来源、协议
评分权重

创新价值 35% + 功能实现 45% + 文档 10% + 视频/PPT 10%。功能实现权重最大，必须保证5个功能完整可运行；加分项是拉开差距的关键。

# backend/app/llm/spark_provider.py
from sparkai.llm.llm import ChatSparkLLM
from sparkai.core.messages import ChatMessage
from langchain_core.language_models import BaseChatModel
from langchain_core.messages import AIMessage, HumanMessage, SystemMessage
from langchain_core.outputs import ChatGeneration, ChatResult
from typing import List, Optional, Iterator, AsyncIterator
import os

class SparkProvider(BaseChatModel):
    """讯飞星火大模型 Provider — 适配 LangChain 接口"""

    def __init__(self, domain: str = None, temperature: float = 0.7, **kwargs):
        super().__init__(**kwargs)
        self._spark = ChatSparkLLM(
            spark_api_url=f"wss://spark-api.xf-yun.com/{domain or os.getenv('SPARK_DOMAIN', '4.0Ultra')}/chat",
            spark_app_id=os.getenv("SPARK_APPID"),
            spark_api_key=os.getenv("SPARK_API_KEY"),
            spark_api_secret=os.getenv("SPARK_API_SECRET"),
            spark_llm_domain=domain or os.getenv("SPARK_DOMAIN", "4.0Ultra"),
            temperature=temperature,
            streaming=True,
        )

    @property
    def _llm_type(self) -> str:
        return "spark"

    def _generate(self, messages: List[BaseMessage], stop=None, **kwargs):
        spark_msgs = [ChatMessage(role=self._map_role(m), content=m.content)
                      for m in messages]
        response = self._spark.generate([spark_msgs])
        # 解析响应...
        return ChatResult(generations=[...])

    @staticmethod
    def _map_role(msg: BaseMessage) -> str:
        if isinstance(msg, SystemMessage): return "system"
        if isinstance(msg, HumanMessage): return "user"
        if isinstance(msg, AIMessage): return "assistant"
        return "user"

    _instances = {}

    @staticmethod
    def get_for_stage(stage: str) -> BaseChatModel:
        """根据阶段返回对应模型实例"""
        routes = json.loads(os.getenv("MODEL_ROUTES", "{}"))
        model_spec = routes.get(stage, os.getenv("DEFAULT_MODEL", "spark:spark-4.0-ultra"))
        return LLMFactory._get_or_create(model_spec)

    @staticmethod
    def _get_or_create(spec: str) -> BaseChatModel:
        if spec in LLMFactory._instances:
            return LLMFactory._instances[spec]

        provider, model = spec.split(":", 1)
        if provider == "spark":
            instance = SparkProvider(domain=model, temperature=0.7)
        elif provider == "openai":
            from langchain_openai import ChatOpenAI
            instance = ChatOpenAI(model=model)
        # ... 其他 Provider
        LLMFactory._instances[spec] = instance
        return instance

# backend/app/orchestration/graph.py
from typing import TypedDict, List, Optional, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.state import CompiledStateGraph
import operator

class LearningState(TypedDict):
    """全局状态 — LangGraph 共享上下文"""
    session_id: str
    user_message: str
    # 画像
    student_profile: dict              # 6维度画像
    profile_updated: bool
    # 路径规划
    learning_path: dict                # 路径规划结果
    resource_tasks: List[dict]         # 待生成的资源清单
    # 资源生成 (聚合)
    generated_resources: Annotated[List[dict], operator.add]
    # 辅导
    tutor_response: Optional[str]
    # 评估
    evaluation: Optional[dict]
    # 控制
    next_action: str                   # 下一步动作


async def profile_node(state: LearningState) -> LearningState:
    """节点1：画像构建 — 从对话中抽取学生特征"""
    from app.agents.profile_extractor import ProfileExtractorAgent
    agent = ProfileExtractorAgent()
    profile = await agent.extract(state["session_id"], state["user_message"])
    return {
        **state,
        "student_profile": profile,
        "profile_updated": True,
        "next_action": "plan",
    }

async def planner_node(state: LearningState) -> LearningState:
    """节点2：路径规划 — 分析画像，规划学习路径"""
    from app.agents.path_planner import PathPlannerAgent
    agent = PathPlannerAgent()
    path_result = await agent.plan(
        profile=state["student_profile"],
        request=state["user_message"]
    )
    return {
        **state,
        "learning_path": path_result["path"],
        "resource_tasks": path_result["resource_tasks"],
        "next_action": "generate_resources",
    }

async def resource_doc_node(state: LearningState) -> LearningState:
    """节点3a：文档生成 Agent"""
    from app.agents.doc_agent import DocAgent
    agent = DocAgent()
    result = await agent.generate(
        profile=state["student_profile"],
        task=next(t for t in state["resource_tasks"] if t["type"] == "doc")
    )
    return {"generated_resources": [result]}

async def resource_quiz_node(state: LearningState) -> LearningState:
    """节点3b：题库生成 Agent — 复用 OpenMAIC 测验系统"""
    from app.agents.quiz_agent import QuizAgent
    agent = QuizAgent()
    result = await agent.generate(
        profile=state["student_profile"],
        task=next((t for t in state["resource_tasks"]
                   if t["type"] == "quiz"), None)
    )
    return {"generated_resources": [result]}

# ... reader_node, mindmap_node, video_node, code_node 类似定义

async def evaluator_node(state: LearningState) -> LearningState:
    """节点5：效果评估"""
    from app.agents.evaluator import EvalAgent
    agent = EvalAgent()
    eval_result = await agent.evaluate(
        session_id=state["session_id"],
        profile=state["student_profile"],
        resources=state["generated_resources"],
    )
    return {
        **state,
        "evaluation": eval_result,
        "student_profile": eval_result.get("profile_update",
                                            state["student_profile"]),
        "next_action": "end",
    }


def build_learning_graph() -> CompiledStateGraph:
    """构建学习编排图"""
    builder = StateGraph(LearningState)

    # 添加节点
    builder.add_node("profile", profile_node)
    builder.add_node("planner", planner_node)
    builder.add_node("doc_gen", resource_doc_node)
    builder.add_node("quiz_gen", resource_quiz_node)
    builder.add_node("mindmap_gen", resource_mindmap_node)
    builder.add_node("video_gen", resource_video_node)
    builder.add_node("code_gen", resource_code_node)
    builder.add_node("reading_gen", resource_reading_node)
    builder.add_node("tutor", tutor_node)
    builder.add_node("evaluator", evaluator_node)

    # 定义边
    builder.add_edge(START, "profile")
    builder.add_edge("profile", "planner")

    # 规划完成后 → 并行分发到所有资源生成节点
    builder.add_edge("planner", "doc_gen")
    builder.add_edge("planner", "quiz_gen")
    builder.add_edge("planner", "mindmap_gen")
    builder.add_edge("planner", "video_gen")
    builder.add_edge("planner", "code_gen")
    builder.add_edge("planner", "reading_gen")

    # 所有资源生成完成后 → 辅导
    builder.add_edge("doc_gen", "tutor")
    builder.add_edge("quiz_gen", "tutor")
    builder.add_edge("mindmap_gen", "tutor")
    builder.add_edge("video_gen", "tutor")
    builder.add_edge("code_gen", "tutor")
    builder.add_edge("reading_gen", "tutor")

    # 辅导 → 评估 → 结束
    builder.add_edge("tutor", "evaluator")
    builder.add_edge("evaluator", END)

    return builder.compile()

7.2 RAG 检索接口

# backend/app/knowledge/retriever.py
class KnowledgeRetriever:
    """RAG检索 + 4层防幻觉"""

    def __init__(self):
        self.embeddings = HuggingFaceBgeEmbeddings(model_name="BAAI/bge-m3")
        self.vectorstore = Chroma(
            persist_directory="./chroma_db",
            embedding_function=self.embeddings,
        )

    def search(self, query: str, k: int = 5) -> List[str]:
        """检索最相关的知识片段"""
        docs = self.vectorstore.similarity_search(query, k=k)
        return [doc.page_content for doc in docs]

    def verify_facts(self, generated_text: str) -> dict:
        """第3层防护：事实核查 — 生成后的交叉验证"""
        # 从生成文本中提取关键声明
        claims = self._extract_key_claims(generated_text)
        results = {}
        for claim in claims:
            evidence = self.search(claim, k=3)
            results[claim] = {
                "evidence": evidence,
                "confidence": self._calc_confidence(claim, evidence)
            }
        return results
知识库 RAG 是防幻觉的基础

所有资源生成 Agent 在调用 LLM 时，必须先检索知识库，将检索结果作为上下文提供给 LLM。换言之，LLM 不是"凭空生成"，而是"基于知识库内容重组"。这确保了学术内容的准确性。赛题明确要求"防幻觉"，这是最核心的工程方案。

# 学习路径动态调整（评估Agent触发）
async def adjust_learning_path(state: LearningState) -> LearningState:
    """根据测评结果动态调整路径"""
    evaluation = state["evaluation"]
    current_path = state["learning_path"]

    if assessment["mastery_level"] > 0.8:
        # 掌握良好 → 跳过基础步骤，进入进阶
        current_path = skip_basic_steps(current_path)
    elif assessment["mastery_level"] < 0.6:
        # 掌握不足 → 回退重学
        current_path = insert_review_steps(
            current_path, assessment["weak_points"]
        )

    return {**state, "learning_path": current_path}

11 第8步：智能辅导 + 效果评估（加分项）

为什么一定要做加分项

功能实现占45%，两个加分项在这45%中显著拉开差距。"画像构建→资源生成→路径规划→辅导→评估→画像更新"形成完整闭环，体现"创新价值"。参考 OpenMAIC 已有的多智能体讨论 + 白板 + TTS + 测验评分系统，大部分基础能力可复用。

11.1 智能辅导 Agent

# backend/app/agents/tutor.py
class TutorAgent:
    """智能辅导 Agent — 多模态答疑"""

    async def answer(self, session_id: str, question: str,
                     profile: dict) -> dict:
        llm = LLMFactory.get_for_stage("tutor-chat")
        knowledge = KnowledgeRetriever().search(question, k=5)

        response = await llm.ainvoke(f"""
        你是学习辅导员。基于知识库内容回答学生问题。
        根据学生画像({json.dumps(profile)})调整解答方式。

        知识库上下文: {knowledge}
        学生提问: {question}

        输出JSON:
        {{
          "text": "文字解答",
          "diagram": "Mermaid语法的图解(如果有助于理解)",
          "code": "代码示例(如果是编程问题)",
          "audio_required": true/false
        }}
        """)

        result = json.loads(response.content)

        # 如果学生是听觉型或问题复杂，生成语音讲解
        if result.get("audio_required") or \
           profile.get("cognitive_style") == "auditory":
            tts = SparkTTS()
            result["audio_url"] = await tts.synthesize(result["text"])

        return result

11.2 学习效果评估 Agent

# backend/app/agents/evaluator.py
class EvalAgent:
    """评估 Agent — 5 维度分析"""

    async def evaluate(self, session_id: str,
                       profile: dict, resources: list) -> dict:
        # 收集学习行为数据
        stats = await self._collect_stats(session_id)

        return {
            "overall_score": self._calc_overall(stats),
            "knowledge_mastery": stats["quiz_scores"],     # 题库正确率
            "behavior_analysis": {                          # 行为分析
                "total_time": stats["total_time"],
                "completion_rate": stats["completion_rate"],
                "review_frequency": stats["review_count"],
            },
            "weak_points": self._identify_weaknesses(stats["errors"]),
            "progress_trend": self._calc_trend(stats["history"]),
            "recommendations": self._generate_recommendations(stats),
            "profile_update": self._suggest_profile_update(stats, profile),
        }

13 第10步：四层防幻觉 + 内容安全机制

层级机制技术实现
第1层检索约束 所有 LLM 调用前，先通过 RAG 检索知识库获取准确内容，作为 Prompt 上下文强制约束
第2层Prompt 约束 System Prompt 明确规则："仅基于提供的知识库内容回答，不确定时请说明'当前知识库中暂无此内容'"
第3层事实核查 生成后再次用 RAG 检索，交叉验证关键事实的准确性。对数值、公式、代码做特别检查
第4层引用标注 在文档中标注内容来源（教材章节/页码），便于评审时验证准确性 
# 防幻觉 System Prompt 模板
SYSTEM_PROMPT = """你是教育内容生成助手，严格遵循以下规则：

1. 【知识来源】仅基于提供的【知识库内容】生成学习资源。
   不得编造知识库中不存在的信息。
2. 【不确定声明】如果知识库中没有足够信息回答某个具体问题，
   请明确说明：「当前知识库中暂无此内容，建议查阅其他资料。」
3. 【引用来源】生成内容时必须标注知识来源。
   格式：「（参考：[教材名] 第X章 第Y节）」
4. 【关键内容核查】对于数值、公式、代码等关键内容，
   必须确保与知识库原文一致，不得改动。
5. 【内容边界】仅生成与指定课程知识相关的教育内容，
   不得生成与学习无关的内容。

【知识库内容】：
{retrieved_context}"""
防幻觉在评分中的重要性

赛题原文写明："需设计完善的'防幻觉'与内容安全过滤机制"。这是非功能性需求中的明确要求，如果评审时发现生成的课程内容有事实错误（如公式错误、概念混淆），会严重影响"功能实现及技术要求"45%的评分。

测试类型内容工具

性能优化要点

  • 并行生成：6 个资源 Agent 通过 LangGraph 并行分发，而非串行等待
  • SSE 流式推送：Vercel AI SDK 原生 SSE，用户看到打字机效果而非白屏等待
  • 结果缓存：RAG 检索结果用 Redis 缓存（相同知识点不重复检索）
  • 任务队列：视频生成等长耗时任务用 Celery/Redis Queue 异步处理
