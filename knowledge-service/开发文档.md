## 1. 模块概述

### 1.1 定位

`knowledge-service` 是"基于大模型的个性化资源生成与学习多智能体系统"中负责**知识库管理**的独立模块。它封装了讯飞星火知识库 (ChatDoc) API，通过 REST 接口对外提供语义检索与事实验证能力。

### 1.2 与主应用的关系

```
┌──────────────────────────┐    HTTP REST    ┌──────────────────────────┐
│   Next.js 主应用           │ ←───────────→ │   knowledge-service       │
│                            │                │   (Python FastAPI)        │
│  lib/knowledge/           │  /api/v1/       │                           │
│    retriever.ts ──────────┼──→ search       │   ChatDoc API             │
│    fact-check.ts ─────────┼──→ fact-check   │   星火知识库云端           │
│                            │                │                           │
└──────────────────────────┘                └──────────────────────────┘
```

### 1.3 赛题要求对照

| 赛题要求 | 本模块实现 |
|----------|-----------|
| 讯飞星火知识库 | ✅ ChatDoc API，云端托管 |
| 至少一门完整高校课程 | ✅ 数据结构与算法完整课程 |
| 四层防幻觉 | ✅ 第1层(检索约束) + 第3层(事实核查) |
| 讯飞 AI 工具使用 | ✅ 星火知识库 + 星火大模型双产品 |

## 2. 技术架构

| 层级 | 技术 | 说明 |
|------|------|------|
| Web 框架 | FastAPI 0.115+ | 异步高性能，自动生成 Swagger 文档 |
| 知识库后端 | 讯飞星火知识库 (ChatDoc) | 云端托管，自动分块/向量化/检索 |
| 鉴权 | HMAC-SHA1 签名 | `base64(hmac-sha1(apiSecret, md5(appId+timestamp)))` |
| HTTP 客户端 | httpx | 异步 HTTP 调用 |

## 4. 快速开始

### 4.1 环境要求

- Python 3.11+

### 4.2 启动

```bash
pip install -r requirements.txt    # 仅 4 个轻量依赖，秒装
uvicorn app.main:app --port 8000
```

### 4.3 验证

```bash
# 健康检查
curl http://localhost:8000/api/v1/health

# 语义检索
curl -X POST http://localhost:8000/api/v1/search \
  -H "Content-Type: application/json" \
  -d '{"query": "数组的时间复杂度", "top_k": 3}'

# 事实验证
curl -X POST http://localhost:8000/api/v1/fact-check \
  -H "Content-Type: application/json" \
  -d '{"content": "数组访问时间复杂度为 O(1)，插入为 O(n)", "topic": "数组"}'
```

或者浏览器打开 `http://localhost:8000/docs` 用 Swagger 交互测试。

## 5. API 接口文档

所有接口前缀：`/api/v1`

### 5.1 语义检索 `POST /api/v1/search`

```bash
curl -X POST http://localhost:8000/api/v1/search \
  -H "Content-Type: application/json" \
  -d '{"query":"链表插入删除的时间复杂度","top_k":3}'
```

**请求**：`{"query": "...", "top_k": 5}`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query | string | 是 | 检索查询文本 |
| top_k | int | 否 | 返回数量，默认 5 |

**响应**：`{"results": [{"id": "...", "text": "...", "source": "...", "score": 0.87}]}`

### 5.2 事实验证 `POST /api/v1/fact-check`

```bash
curl -X POST http://localhost:8000/api/v1/fact-check \
  -H "Content-Type: application/json" \
  -d '{"content":"数组访问O(1)，插入O(n)","topic":"数组"}'
```

**请求**：`{"content": "...", "topic": ""}`

**响应**：`{"score": 80, "flagged": ["错误声明..."], "checked": 5}`

### 5.3 文档上传 `POST /api/v1/ingest`

```bash
curl -X POST http://localhost:8000/api/v1/ingest \
  -F "files=@文档.pdf"
```

### 5.4 健康检查 `GET /api/v1/health`

```bash
curl http://localhost:8000/api/v1/health
# → {"status":"healthy","embedding_model":"星火知识库 (ChatDoc)"}
```

## 7. 防幻觉机制

| 层级 | 机制 | 实现 |
|------|------|------|
| 第1层 | 检索约束 | 所有检索先过星火知识库，结果作为 LLM Prompt 上下文 |
| 第2层 | Prompt 约束 | 主应用 `resource-runner.ts` 中限制 |
| 第3层 | 事实核查 | `spark_kb.fact_check()`：提取可核声明 → 回查知识库 → score |
| 第4层 | 引用标注 | 主应用 `sources` 字段标注 |

## 10. 常见问题

**Q: 返回空结果？**
A: 确认星火知识库中已上传文档且状态为"已完成"；确认 `SPARK_KB_REPO_ID` 正确。

**Q: 401 鉴权错误？**
A: 确认 APPID 和 APISecret 是星火知识库服务的凭证（在控制台应用详情中查看）。

**Q: 如何更换知识库？**
A: 修改环境变量 `SPARK_KB_REPO_ID` 为目标知识库 ID。
